1. Executive Summary

This project is an AI-powered case summarization bot that operates inside Webex. Engineers type a ServiceNow case number in a direct message, and the bot returns a structured, concise summary within 5–8 seconds — eliminating the need to open ServiceNow and manually read through long case timelines.

The bot uses Cisco CIRCUIT LLM (gpt-5-nano) for AI summarization, ensuring all data stays within Cisco's approved infrastructure. It is deployed on AWS Lambda as a serverless application with zero infrastructure overhead.

2. Problem Statement

Support engineers spend significant time reading through ServiceNow case histories to understand the current status of a ticket. A typical case may have 10–30+ journal entries, 5–15 customer emails, and multiple internal work notes. This leads to:

  • 15–30 minutes per case spent reading before understanding the situation
  • Context switching — engineers must leave Webex calls to open ServiceNow
  • Repeated work — without quick summaries, engineers repeat already-tried diagnostic steps
  • Delayed response — time spent reading is time not spent resolving
Estimated impact: Engineers spend ~30% of their time reading case history. For a team of 100, this translates to ~500–800 hours/month of lost productivity.

3. Solution Overview

The Case Summarization Agent brings AI-generated summaries directly into Webex, where engineers already collaborate. The flow is:

  1. Engineer sends a case number (e.g., CS0001026) to the bot in Webex
  2. Bot instantly displays a "Generating summary…" interactive card (<2 seconds)
  3. Bot fetches case data, journal entries, and email history from ServiceNow CSM API
  4. Builds a chronological timeline and sends it to CIRCUIT LLM for summarization
  5. Delivers a structured summary as an Adaptive Card in Webex (5–8 seconds total)

Summary Structure

The AI generates a clean, engineer-focused summary with these sections:

SectionDescription
ProblemWhat is broken and who is affected (1–2 sentences)
Root CauseIf identified in work notes (1 sentence)
What Was DoneActions taken by engineers, deduplicated
Current StatusWhere the ticket stands now
Next StepsIf explicitly mentioned in the timeline

Key Design Decisions

  • Async self-invocation: The webhook handler returns immediately (<2s) after sending the "working" card, then invokes the same Lambda asynchronously to run the heavy pipeline. This avoids API Gateway's 30-second timeout.
  • Vendor path isolation: Third-party packages are stored in a vendor/ directory with sys.path bootstrapping, keeping the project root clean.
  • Adaptive Cards: Rich interactive UI in Webex with bold section headers, metadata display, and action buttons.

8. CIRCUIT LLM Integration

The bot uses Cisco's internal CIRCUIT API gateway to access LLM models:

  • Token acquisition: OAuth2 client_credentials flow via id.cisco.com
  • Model: gpt-5-nano (approved under Free Fair Use tier)
  • Prompt engineering: Custom prompt designed to produce concise, factual, deduplicated summaries under 200 words
  • Data classification: Cisco Confidential (as per CIRCUIT registration)
  • App Key: Registered under Department ID 123212180 (CX)
